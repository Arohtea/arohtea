# Prompt Engineering 实战指南

大语言模型本质上是一个极其复杂的"文字接龙"系统。它的输出，90% 取决于你给它的输入——也就是 **Prompt（提示词）**。写好 Prompt 是一项真正的工程技能。

## 基本原则一：给角色，设场景

不加任何上下文直接提问，会得到平庸的通用回答。

```
❌ 糟糕的 Prompt:
"翻译这段话"

✅ 好的 Prompt:
"你是一个专业的技术文档翻译员，精通中英文，尤其熟悉软件工程领域的术语。
请将以下英文技术文档翻译为简体中文，保持专业语气，专有名词保留英文原文。
原文：{text}"
```

## 基本原则二：给示例（Few-Shot）

与其用语言描述你想要的格式，不如直接给几个例子。

```
将用户评论分类为：正面 / 中性 / 负面

示例：
- "产品质量很好，下次还会买！" → 正面
- "一般般吧，没什么特别的。" → 中性
- "包装破损，强烈不推荐！" → 负面

现在请分类：
- "物流比预期快，但东西有点贵。" → ?
```

## 基本原则三：思维链（Chain of Thought）

复杂推理任务中，加一句 *"请一步一步思考"* 可以显著提升模型的准确率。这迫使模型在给出最终答案前先进行过程推演。

```
请解决以下数学题，并一步一步展示你的推理过程：
如果一列火车以 120km/h 的速度行驶，2.5 小时后距离起点多远？
```

## 结构化输出：让 AI 返回 JSON

在工程实践中，我们经常需要 AI 返回结构化数据而非自然语言：

```
你是一个信息提取引擎。请从以下文本中提取关键实体，并以 JSON 格式返回：

{
  "people": [],
  "organizations": [],
  "dates": []
}

文本：2024年，李雷加入了阿里巴巴集团，担任技术总监。
```

> 配合 `response_format: { type: "json_object" }` 参数（OpenAI API），可以强制模型输出合法的 JSON，避免解析错误。

掌握这些原则后，Prompt 工程师不再只是"会聊天的人"，而是真正的 AI 系统设计师。
